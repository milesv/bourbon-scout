# Bourbon Scout

Allocated bourbon inventory scraper with Discord webhook alerts. Monitors 7 retailers for 38 bottles (37 rare/allocated + 1 canary health check) near a zip code, sends per-store Discord alerts with @here pings when bottles are found, plus a quiet summary with per-scraper health metrics after each scan.

## Architecture

- **`scraper.js`** ‚Äî Main entry point. Contains all scrapers, Discord alerting, and poll orchestration. Schedule-aware poll loop: 30 min default, 20 min boost (Tue/Thu nights), active 5 PM ‚Äì 10 AM MT only. ¬±3 min jitter. Single-run via `RUN_ONCE=true` for CI. Scans all stores concurrently with a limit of 8.
- **`com.bourbon-scout.plist`** ‚Äî macOS launchd daemon config. Keeps scraper running with auto-restart. Logs to `~/Library/Logs/bourbon-scout.log`.
- **`scripts/install-daemon.sh`** / **`scripts/uninstall-daemon.sh`** ‚Äî Install/remove the launchd daemon.
- **`browser-profiles/`** ‚Äî Per-retailer persistent Chrome profile directories (auto-created). Contains HTTP cache, service workers, IndexedDB, visited history. Gitignored.
- **`lib/geo.js`** ‚Äî Zip-to-coordinates (`zipToCoords`) via zippopotam.us API and `haversine` distance calculation. No dependencies.
- **`lib/discover-stores.js`** ‚Äî Auto-discovers nearby stores for all retailers given a zip code and radius. Caches results to `stores.json` with a 7-day TTL. Uses Playwright for browser-based store locators and Kroger REST API for Kroger.
- **`lib/fallback-stores.js`** ‚Äî Static hardcoded store data for the 85283 area. Used when browser-based store locators fail (e.g., CI datacenter IPs get blocked).
- **`state.json`** ‚Äî Per-store stock state, nested as `{ "retailer": { "storeId": { "bottles": { "name": { url, price, sku, firstSeen, lastSeen, scanCount } }, "lastScanned" } } }`. Auto-generated, safe to delete for a fresh start.
- **`stores.json`** ‚Äî Cached store discovery results. Auto-generated, safe to delete to force re-discovery.
- **`.github/workflows/bourbon-scout.yml`** ‚Äî GitHub Actions cron job (currently disabled in favor of local daemon). Schedule: 30 min default, 20 min on Tue/Thu nights (Wed/Fri UTC), active 5 PM ‚Äì 10 AM MST only. Self-hosted Mac runner (`bourbon-scout-mac`). Caches state files between runs. Uses `RUN_ONCE=true`. 25-minute timeout.
- **`debug/`** ‚Äî Screenshots, HTML dumps, and CSS selector lists captured from retailer store locator pages. Used for fixing selectors when sites change. Generated by `debug-locators.js`.

## Key Patterns

- **ESM throughout** ‚Äî `"type": "module"` in package.json, all files use `import`/`export`
- **Clean browser mode (all scrapers)** ‚Äî All 5 browser scrapers use `launchRetailerBrowser(key, { clean: true })` which launches Chrome WITHOUT stealth plugin or rebrowser-patches CDP interception. Uses raw `rebrowserChromium.launchPersistentContext()` instead of the `addExtra()`-wrapped version. **Key discovery**: stealth+rebrowser CDP modifications are fingerprinted by both PerimeterX and Akamai as automation signals ‚Äî removing them makes the browser undetectable. Defaults to headed on Mac (`headless: !IS_MAC`) ‚Äî the underlying `rebrowser-playwright-core` still has rebrowser-patches in its CDP layer, which headless mode exposes more than headed. On Mac, launches system Chrome (`/Applications/Google Chrome.app`) via `executablePath` for authentic TLS fingerprint. Falls back to bundled Chromium on other platforms. Launches with `--disable-blink-features=AutomationControlled` and `ignoreDefaultArgs: ['--enable-automation']`. Browser contexts get platform-aware Chrome 145 UA + `Sec-CH-UA` Client Hints matching `FETCH_HEADERS`. **Minimized on Mac** ‚Äî CDP `Browser.setWindowBounds` minimizes the window after launch so it doesn't steal focus during scans.
- **rebrowser-patches + Stealth (store discovery only)** ‚Äî `rebrowser-playwright-core` wrapped with `playwright-extra`'s `addExtra()` and `puppeteer-extra-plugin-stealth` is still used for `discover-stores.js` and the shared `newPage()` browser, but NOT for any retailer scraper browser. All 5 retailer scrapers use clean mode instead.
- **PerimeterX challenge solver** ‚Äî `solveHumanChallenge(page)` detects and solves PerimeterX "Press & Hold" challenges. Checks `document.body.innerText` for "Press & Hold" text, finds `#px-captcha` element (PerimeterX renders the button via captcha.js inside this container), moves mouse naturally to center, holds for 8-12s, releases, waits for navigation. Called after homepage pre-warm and on blocked search pages for Total Wine, Costco, and Walgreens. Only works when captcha.js renders the interactive button (requires clean browser mode).
- **Per-retailer browser mutex** ‚Äî `acquireRetailerLock(key)` serializes browser access per retailer via promise chains. Prevents multiple stores from opening concurrent pages on the same persistent context, which anti-bot systems flag as automated behavior. Different retailers can run concurrently.
- **Browser fail-fast** ‚Äî `retailerBrowserBlocked[key]` flag set when first store's browser attempt fails. Remaining stores skip browser entirely since same IP/profile means they'll fail too. Reset each poll.
- **Platform-aware User-Agent** ‚Äî `IS_MAC` detection at module level. macOS Chrome UA + `Sec-CH-UA-Platform: "macOS"` on Mac, Windows Chrome on Linux/CI. Prevents UA-vs-TLS fingerprint mismatch that bot detectors flag.
- **Lazy browser launch** ‚Äî Browser is only launched when a scraper actually needs it (via `newPage()`). When all scrapers succeed via fetch-first, no browser is started.
- **Atomic state writes** ‚Äî `saveState()`, `saveBrowserState()`, `saveCache()` (discover-stores) all write to `.tmp` then `rename()` (atomic on POSIX). Prevents corruption on crash mid-write.
- **Browser cookie persistence** ‚Äî `loadBrowserState()`/`saveBrowserState()` persist Playwright `storageState` to `browser-state.json`. In-memory cache (`browserStateCache`) avoids re-reading from disk on every `newPage()`. Cookies accumulate across polls, reducing bot detection signals.
- **Parallel query execution** ‚Äî API scrapers (Kroger, Safeway) run all 12 queries via `Promise.all` with 50ms staggered starts. Fetch scrapers (Costco, TotalWine, Walmart) use batched parallelism via `runWithConcurrency(queryTasks, 2)`. Sam's Club uses `runWithConcurrency(productTasks, 2)`.
- **Kroger pagination** ‚Äî Fetches page 2 (`filter.start=50`) when first page returns exactly 50 results, catching allocated bottles that spill past the first page. Same pattern used by Safeway. Page 2 failures are isolated in their own try/catch ‚Äî a page 2 JSON parse error logs a warning but doesn't lose page 1 data or mark the query as failed.
- **Kroger null-inventory guard** ‚Äî Requires `stockLevel != null` in addition to checking for `TEMPORARILY_OUT_OF_STOCK`. Prevents false positives when Kroger API returns products with null/missing inventory data.
- **Concurrent scanning** ‚Äî All stores run concurrently via `runWithConcurrency(tasks, 8)`. Browser and API scrapers are unified in the same task queue.
- **Structured data extraction over CSS selectors** ‚Äî Scrapers prefer embedded JSON (Walmart `__NEXT_DATA__`, Total Wine `window.INITIAL_STATE`) over fragile CSS selectors. Costco uses stable MUI `data-testid` attributes. CSS selector fallbacks exist but are last resort.
- **Residential proxy support** ‚Äî Optional `PROXY_URL` env var routes all scraper traffic (browser + fetch) through a proxy. Supports HTTP/HTTPS and SOCKS5/SOCKS4 protocols ‚Äî `createProxyAgent()` auto-detects based on URL scheme. Playwright browser proxy requires URL parsing into `{ server, username, password }` (can't pass the full URL). Discord webhooks skip the proxy. With proxy set, Walmart fetch-first works on CI too.
- **Sticky proxy sessions** ‚Äî `refreshProxySession()` called at the start of each `poll()`. For DataImpulse, switches to a random sticky port (10000-20000) so all requests in a scan share one residential IP for ~30 min. Prevents IP rotation from breaking cookie chains (e.g., Costco homepage pre-warm cookies tied to one IP, search requests from another).
- **Mid-poll proxy rotation** ‚Äî `rotateRetailerProxy(key)` switches a retailer to a fresh sticky port (= new residential IP) when the current IP is detected as burned. Triggered automatically after 2 consecutive fetch failures in Costco, Walmart, and Sam's Club fetch paths. Clears stale cookies from the old IP's session. Proxy URL is read dynamically per-request (`getRetailerProxyUrl()`) so rotation takes effect immediately for remaining queries.
- **Per-retailer IP isolation** ‚Äî Each retailer gets its own sticky proxy port (= its own residential IP) via `retailerProxyAgents`. The general `proxyAgent` uses a separate port (`basePort + retailers.length`) to avoid sharing with any retailer. Prevents cross-retailer IP flagging cascades ‚Äî if Costco's IP gets burned, Walmart's is unaffected. `getRetailerProxy(key)` returns the per-retailer fetch agent; `getRetailerBrowserProxy(key)` returns Playwright-compatible `{ server, username, password }`.
- **Persistent browser profiles** ‚Äî `launchRetailerBrowser(retailerKey)` uses `chromium.launchPersistentContext(profileDir)` to maintain full Chrome profiles per retailer in `browser-profiles/{key}/`. HTTP cache, service workers, IndexedDB, and visited history persist on disk across polls ‚Äî the scraper looks like a returning visitor. Within a poll, `retailerBrowserCache` shares the context across stores. Pages are closed after each use; contexts closed at end of poll via `closeRetailerBrowsers()`. Profile data survives context close for next poll. On launch, stale `SingletonLock`/`SingletonSocket`/`SingletonCookie` files are auto-cleaned to prevent hangs after Chrome crashes.
- **Browser launch failure isolation** ‚Äî All 5 wrapper functions (`scrapeCostcoStore`, `scrapeTotalWineStore`, `scrapeWalmartStore`, `scrapeWalgreensStore`, `scrapeSamsClubStore`) catch `launchRetailerBrowser` failures with try/catch. On failure: logs error, tracks health as "fail", returns `[]`. Prevents a single Chrome crash from taking down the entire poll via `runWithConcurrency`.
- **Walgreens retry on block** ‚Äî Since Walgreens is `scrapeOnce`, a single Akamai challenge wipes out all stores. `scrapeWalgreensStore` retries once with a fresh browser context if ‚â•50% of queries were blocked on the first attempt. Clears `retailerBrowserCache` and adds a 3-5s cooldown between attempts.
- **Query rotation across scans** ‚Äî `getQueriesForScan(SEARCH_QUERIES)` alternates which queries run based on `scanCounter % 2`. Each scan fires ~7 queries instead of 13 (half the non-canary queries + always the canary). All 38 bottles are covered within 2 consecutive scans (60 min at 30-min intervals). Reduces per-session "hammering" that bot detection ML flags.
- **Query order randomization** ‚Äî `shuffle(getQueriesForScan(SEARCH_QUERIES))` via Fisher-Yates before every scraper loop to avoid predictable access patterns that anti-bot ML models flag.
- **Chrome TLS fingerprint impersonation (got-scraping)** ‚Äî All scraper fetch paths use `got-scraping` from Apify instead of `node-fetch`. `got-scraping` impersonates Chrome's TLS handshake (JA3/JA4 fingerprint, cipher suites, extensions order), making HTTP requests indistinguishable from real Chrome at the network layer. `scraperFetch()` wraps `gotScraping()` with a node-fetch-compatible response interface (`{ ok, status, headers, text(), json() }`). `scraperFetchRetry()` adds one-retry-on-error. API scrapers (Kroger, Safeway) and Discord webhooks still use raw `node-fetch` (no bot detection). Proxy passed as `proxyUrl` string option (not `agent` object).
- **Fetch-first with browser fallback** ‚Äî 4 of 5 browser-based scrapers try `got-scraping` fetch first (with Chrome TLS fingerprint) before spinning up Playwright. Walmart parses `__NEXT_DATA__`, Sam's Club parses `__NEXT_DATA__` per-product, Costco parses `data-testid` tiles via cheerio. Falls back to clean browser instances if blocked. Total Wine skips fetch entirely (PerimeterX always blocks ‚Äî requires JS sensor execution) and goes straight to clean browser. Fetch paths require ‚â•25% of queries to succeed (`validPages >= ceil(totalQueries/4)`) before trusting results ‚Äî prevents a single valid page from suppressing browser fallback when most queries are blocked.
- **Per-product-URL scraping** ‚Äî Sam's Club search excludes spirits, so `SAMSCLUB_PRODUCTS` maps bottle names to product IDs. Each product page is fetched individually at `/ip/{productId}` instead of using SEARCH_QUERIES. Only bottles with known Sam's Club pages are checked.
- **Fetch retry** ‚Äî `fetchRetry()` (for `node-fetch` API paths) and `scraperFetchRetry()` (for `got-scraping` scraper paths) retry once on network errors with 1s backoff and `console.warn` on first failure.
- **Brace-counting JSON extraction** ‚Äî Total Wine INITIAL_STATE and Walmart `__NEXT_DATA__` are both extracted from HTML using a character-by-character brace counter that tracks string context and escape characters, safely handling `</script>` inside JSON string values.
- **Scrape-once pattern** ‚Äî Costco search has no store filter, so results are identical across warehouses. Walgreens uses `USER_LOC` cookie with zip coordinates ‚Äî nearby stores share results. Sam's Club product pages show availability for the nearest club. All three scraped once and broadcast to all stores.
- **Broad search queries** ‚Äî 12 queries cover all 37 allocated bottles + 1 canary query = 13 total. API scrapers (Kroger, Safeway) use these instead of per-bottle queries (~60% fewer API calls).
- **Canary bottle (Buffalo Trace)** ‚Äî Always-in-stock bottle used as a scraper health check. Found canary = scraper is working. Missing canary = scraper may be broken. Canary results are filtered out before alerts/state ‚Äî only shown in the health section of the summary embed. Canary is also excluded from the OOS list in Discord embeds (`TARGET_BOTTLES` filtered by `CANARY_NAMES` in `buildStoreEmbeds`).
- **Per-scraper health tracking** ‚Äî `scraperHealth` module-level object tracks queries succeeded/failed/blocked per retailer per poll. `trackHealth(retailerKey, outcome)` called in every scraper's query loop (including early-exit paths like Walgreens coords failure). Reset each poll. Summary embed always shows all 7 retailers as inline Discord fields (3 per row): ‚úÖ/‚öÝÔ∏è/‚ùå for retailers with data, ‚è≠Ô∏è for skipped/missing ones.
- **Shared Kroger OAuth token** ‚Äî Fetched once per poll via singleton promise (`fetchRetry` for network resilience), shared across all Kroger stores. Clears cached token on 401 for automatic re-auth. Uses promo price when available (falls back to regular).
- **Per-scraper browser timeout with health tracking** ‚Äî `withTimeout(promise, 180000, null)` wraps all browser scraper calls (Costco, Total Wine, Walmart, Walgreens, Sam's Club). On timeout (3 min), wrappers log a warning, call `trackHealth(key, "blocked")`, and return `[]`. This ensures adaptive retailer skipping engages for completely hung browsers (Akamai challenge pages that never resolve), rather than silently treating them as empty results.
- **Homepage pre-warm with humanization** ‚Äî All browser scrapers (Costco, Total Wine, Walmart, Walgreens, Sam's Club) visit the retailer homepage before searching. Uses `networkidle` wait + 3-5s randomized dwell time to let anti-bot sensor scripts (Akamai `_abck`, PerimeterX `_px*`) fully execute and phone home. After dwell, `humanizePage(page)` scrolls down via mouse wheel events, hovers over a random link, then scrolls back up using wheel events (not `scrollTo` ‚Äî that's a JS teleport detectable by anti-bot sensors). Costco fetch path also pre-warms, collecting `Set-Cookie` headers for subsequent requests.
- **Category navigation flow** ‚Äî After homepage pre-warm, browser scrapers navigate to a spirits/liquor category page (`navigateCategory(page, key)`) before searching. This simulates a real user flow: homepage ‚Üí browse category ‚Üí search. `CATEGORY_URLS` maps retailer keys to known category page URLs (e.g., Costco `/liquor.html`, Total Wine `/spirits/c/spirits`). Non-critical ‚Äî failures are caught and searching continues.
- **Staggered retailer start times** ‚Äî In `poll()`, each retailer's first task sleeps 10-30s (randomized) before starting, so all 7 retailers don't hit the internet simultaneously. First retailer starts immediately; subsequent ones stagger.
- **Direct product URL checks** ‚Äî `loadKnownProducts(state)` extracts previously found bottle URLs from `state.json` at the start of each poll. `checkWalmartKnownUrls(store)` checks known Walmart product pages directly before running the full search ‚Äî less suspicious than searching and catches bottles search might miss.
- **Schedule-aware scanning** ‚Äî Active 5 PM ‚Äì 10 AM MT only (sleeps during work hours). 30 min default, 20 min "boost" on Tue/Thu nights into Wed/Fri mornings. `getMTTime()` uses `Intl.DateTimeFormat` with `America/Phoenix` timezone. `isActiveHour()`/`isBoostPeriod()` determine interval. ¬±3 min jitter per cycle. `parsePollIntervalMs()` still parses `POLL_INTERVAL` as fallback for `RUN_ONCE` mode.
- **Local daemon** ‚Äî `com.bourbon-scout.plist` launchd config with `KeepAlive: true`. `scripts/install-daemon.sh` / `scripts/uninstall-daemon.sh` for setup. Logs to `~/Library/Logs/bourbon-scout.log`.
- **Adaptive retailer skipping** ‚Äî `shouldSkipRetailer(key)` / `recordRetailerOutcome(key, success)` track consecutive failures per retailer. After 3 consecutive polls with <25% query success rate, the retailer is skipped for 30 min to let IP reputation recover. Cooldown auto-expires.
- **Human-like inter-query pacing** ‚Äî Browser scrapers wait 2-4s (randomized) between search queries. All fetch scrapers (Costco, Walmart, Sam's Club) wait 1.5-3s between queries. Real humans take 5-15s between searches, so this interval is still faster but much less suspicious than sub-second bursts.
- **Fetch-only-ok health tracking** ‚Äî Fetch paths (Costco, Walmart, Total Wine, Sam's Club) only track health on success (`trackHealth(key, "ok")`), not on failure. When fetch fails and browser fallback runs, only browser events appear in health metrics. Prevents double-counting inflation (e.g., "0/69" from fetch+browser both tracking failures for 5 Walmart stores).
- **Backup proxy provider** ‚Äî `BACKUP_PROXY_URL` + `BACKUP_PROXY_RETAILERS` env vars let specific retailers use a different proxy provider (e.g., IPRoyal for Costco if DataImpulse IPs are burned at Akamai). Each backup retailer gets its own sticky port on the backup proxy. Unused retailers stay on the primary `PROXY_URL`.
- **Costco fetch resilience** ‚Äî `scrapeCostcoViaFetch` has layered anti-bot countermeasures: (1) cookie pre-warm from homepage, (2) adaptive concurrency (1 for SOCKS5, 4 for HTTP), (3) inter-query jitter (250-500ms), (4) expanded blocked detection via `isCostcoBlocked()` with 8 Akamai patterns, (5) retry-with-backoff on blocked responses (2-3s pause before retry).
- **Bot detection** ‚Äî `isBlockedPage()` checks page title AND body text for challenge/CAPTCHA/access-denied/security-check indicators. Browser scrapers skip queries that hit bot detection instead of recording false "empty" results. Costco fetch path uses `isCostcoBlocked()` with additional Akamai-specific patterns (`_ct_challenge`, `Request unsuccessful`, `Incapsula`, `Enable JavaScript`, `verify you are human`).
- **Per-retailer Sec-Fetch-Site** ‚Äî First query per scraper sends `cross-site` + Google Referer (simulating search click). Subsequent queries send `same-origin` + retailer domain Referer (simulating internal navigation). Uses `.map()` index (`i > 0`) instead of runtime flag to avoid race conditions in concurrent task execution.
- **Chrome-accurate FETCH_HEADERS** ‚Äî `Accept` header matches real Chrome 145 (includes `image/avif`, `image/webp`, `application/signed-exchange`). `Sec-Fetch-User: ?1` header included. `Accept-Encoding` is `gzip, deflate` only (Brotli removed for compatibility).
- **State pruning** ‚Äî `pruneState()` removes stale retailer/store entries from `state.json` on each poll, keeping state clean when stores change.
- **Text normalization** ‚Äî `normalizeText()` converts Unicode curly quotes/apostrophes to ASCII before matching, preventing silent misses when retailers use `\u2019` in product names.
- **Fetch sanity checks** ‚Äî Walmart fetch path validates `__NEXT_DATA__` contains a real `searchResult` structure before trusting results. Prevents challenge pages with valid JSON but no search data from being treated as "nothing in stock."
- **AbortSignal timeouts** ‚Äî All fetch calls (Discord 10s, zipToCoords 10s, Kroger/Safeway/Walmart 15s, Kroger store locator 15s) use `AbortSignal.timeout()` to prevent hung requests from blocking concurrency slots.
- **Error isolation** ‚Äî `runWithConcurrency` wraps tasks in `.catch()` so one failing task never aborts remaining tasks. Costco broadcast loop has per-store try/catch. `saveState`/`closeBrowser`/summary send are all individually error-handled.
- **Store discovery is separate from scraping** ‚Äî `discover-stores.js` handles finding stores, `scraper.js` handles checking inventory
- **State change tracking** ‚Äî `computeChanges()` diffs previous and current scan results. `updateStoreState()` persists `firstSeen`, `lastSeen`, `scanCount` per bottle per store. Re-alerts every N scans via `REALERT_EVERY_N_SCANS`.
- **Color-coded Discord embeds** ‚Äî Green `@here` for new finds, orange for OOS losses, blue re-alerts, purple scan summary. Summary includes scanned store list (grouped by retailer) when nothing is found. 429 rate limit retry with `Retry-After` backoff (throws after 3x 429 to prevent silent alert loss). Embed titles truncated at 256 chars, descriptions at 4096 chars.
- **Google Maps links** ‚Äî Store addresses are hyperlinked to Google Maps universal search URLs.

## Environment Variables

All config is in `.env`:
- `ZIP_CODE` ‚Äî center point for store discovery (currently 85283 / Tempe, AZ)
- `SEARCH_RADIUS_MILES` ‚Äî how far to search (default 15)
- `MAX_STORES_PER_RETAILER` ‚Äî caps per chain (default 5)
- `POLL_INTERVAL` ‚Äî fallback scan interval as cron expression (default `*/30 * * * *`). Only used in `RUN_ONCE` mode; daemon mode uses schedule-aware logic (30 min default, 20 min boost).
- `REALERT_EVERY_N_SCANS` ‚Äî re-alert interval for bottles still in stock (default 4)
- `KROGER_CLIENT_ID`, `KROGER_CLIENT_SECRET` ‚Äî Kroger API OAuth credentials
- `SAFEWAY_API_KEY` ‚Äî Safeway product search API key
- `DISCORD_WEBHOOK_URL` ‚Äî Discord webhook for alerts
- `PROXY_URL` ‚Äî Residential proxy URL. Supports HTTP (`http://user:pass@gw.dataimpulse.com:823`) and SOCKS5 (`socks5://user:pass@host:1080`). Routes browser and fetch scraper traffic through proxy. Discord webhooks are NOT proxied. DataImpulse sticky sessions use ports 10000-20000 (auto-configured by `refreshProxySession()`).
- `BACKUP_PROXY_URL` ‚Äî Optional backup proxy URL (e.g., IPRoyal). Used by retailers listed in `BACKUP_PROXY_RETAILERS`.
- `BACKUP_PROXY_RETAILERS` ‚Äî Comma-separated list of retailer keys to route through `BACKUP_PROXY_URL` instead of `PROXY_URL` (e.g., `costco,totalwine`).
- `RUN_ONCE` ‚Äî Set to `true` for single-run mode (used by GitHub Actions)

## Debugging Store Locators

When a retailer's store locator stops finding stores (selectors broke):
1. Run `node debug-locators.js` to capture fresh screenshots + HTML + selector lists to `debug/`
2. Compare `debug/{retailer}-selectors.txt` against the selectors in `lib/discover-stores.js`
3. Update the locator function, delete `stores.json`, and re-run

## Gotchas

- Costco's search URL (`/s?keyword=`) returns identical results regardless of warehouse ‚Äî that's why we scrape once and broadcast.
- Total Wine uses PerimeterX (HUMAN Security) ‚Äî fetch path always blocked (PerimeterX requires JS sensor execution). Goes straight to clean browser. `got-scraping` fetch function retained but disabled in the wrapper.
- **stealth plugin + rebrowser-patches CAUSE detection** ‚Äî PerimeterX and Akamai fingerprint the CDP modifications these tools make. Plain Chrome with `--disable-blink-features=AutomationControlled` passes undetected. All 5 browser scrapers use `clean: true` mode (raw chromium, no addExtra/stealth). The stealth-wrapped chromium is only used for store discovery (`discover-stores.js`).
- Walmart fetch-first gets blocked on CI (datacenter IPs) but works locally. With `PROXY_URL` set, fetch-first works on CI too. Without proxy, fetch path skips on CI and falls back to browser.
- Walmart fetch validates `__NEXT_DATA__` has a `searchResult` key ‚Äî challenge pages with valid JSON but empty search data correctly fall back to browser instead of reporting "nothing found."
- Store locator CSS selectors in `discover-stores.js` break when retailer sites update their DOM. Kroger's API locator is the most stable.
- Safeway's store locator is at `local.safeway.com/search.html` ‚Äî NOT `safeway.com/stores/search` (that 404s).
- Deleting `state.json` is safe but will send a full report for all stores on next poll.
- The Safeway scraper and Kroger scraper silently skip if their API keys aren't set.
- `stores.json` cache invalidates on zip/radius change or after 7 days.
- `lib/fallback-stores.js` provides static store data when browser-based locators fail on CI.
- Walgreens has no embedded JSON (`__NEXT_DATA__`, `INITIAL_STATE`) ‚Äî server-rendered HTML only. Akamai blocks direct fetch (403), so it's browser-only with no fetch-first path. Store context set via `USER_LOC` cookie (base64 JSON: `{"la":"<lat>","lo":"<lng>","uz":"<zip>"}`).
- Sam's Club search (`samsclub.com/search?q=`) does NOT return spirits ‚Äî product pages exist at `/ip/{slug}/{id}` but are excluded from search results. The scraper uses `SAMSCLUB_PRODUCTS` (a static map of bottle‚ÜíproductId) instead of `SEARCH_QUERIES`. Not all 38 bottles have Sam's Club pages ‚Äî only 8 are currently mapped.
- NordVPN SOCKS5 proxy (`phoenix.us.socks.nordhold.net:1080`) doesn't work for scraping ‚Äî it caps concurrent connections, causing `ConnectionRefused`/`ERR_SOCKS_CONNECTION_FAILED` under load.
- `PROXY_URL` supports both HTTP and SOCKS5 via `createProxyAgent()` auto-detection, but SOCKS5 proxies with connection limits (like NordVPN) will fail under concurrent load. Use a residential HTTP proxy (DataImpulse at $1/GB recommended).
- DataImpulse sticky sessions are **port-based** (10000-20000), NOT username-suffix-based. Appending `-sessid-{id}` to the username breaks authentication. `refreshProxySession()` handles this automatically.
- Playwright requires proxy credentials as separate `{ server, username, password }` fields ‚Äî passing the full URL (like `HttpsProxyAgent` accepts) causes `ERR_INVALID_AUTH_CREDENTIALS`.
- `got-scraping` response API differs from `node-fetch`: `statusCode` (not `status`), `body` as string (not stream), `proxyUrl` option (not `agent`), `timeout: { request: ms }` (not `AbortSignal`). The `scraperFetch()` wrapper normalizes this to a node-fetch-compatible interface.
- `got-scraping` `useHeaderGenerator` is set to `false` in `scraperFetch()` ‚Äî we supply our own Chrome 145-accurate `FETCH_HEADERS` rather than relying on got-scraping's auto-generated headers.
